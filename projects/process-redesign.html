Business Process Manager App Redesign

A heuristic evaluation and design solution iteration

Overview

This product redesign proposal was conducted out of self-interest in my own time for my previous work term. I conducted this study to explore avenues in which mapping of data, information access cost, and the overall experience of the application in question could be improved. The application discussed is a business process management system which provides a number of process templates to achieve some business goal. The templates are composed of reusable “blocks” which are classified as automated (code driven) or manual (requiring human intervention).

Problem

From a user experience standpoint, the application considered can become incredibly complex as several blocks are used in tandem to construct varying business processes. For especially convoluted processes, it becomes necessary to present a user interface that provides the information in a coherent manner so that the state of a process can be quickly discerned.

The application itself had not involved any designers throughout the development process and thus the UI/UX focus is lacking in many parts of the project.

In the current application, users refer to the processes page in which they select the active process of interest from a list of processes. Figure 1 below shows the possible options. The problem lies in the fact that the screen shows fragmented bits of information leaving the onus on the user to make sense of everything.

User Journey

There are two primary user personas to consider: the process administrator and the task worker. The process administrator user was focused because the role has more complicated patterns of application usage. Figure 2 demonstrates some of the major journey breakdowns. The focus of this case study was to enhance the understanding of an active process state.

Information about the different personas came from qualitative data the team had about client business and user requirements.

Heuristic Evaluation

Before considering solutions, an initial heuristic evaluation was conducted on the current application state. The main principles referenced came from Wickens’ principles from An Introduction to Human Factors Engineering and Norman’s principles of interaction design from The Design of Everyday Things.

From the evaluation conducted, it was found that the current application demonstrates weak mapping of data between the process history and process diagram. The process diagram is a static image which cannot be interacted with restricting the user from directly manipulating the diagram to query for information. The two sets of data are tightly coupled, but the UI fails to demonstrate such relationship.

The application also violates attentional principles of display, namely the increase in information access cost due to a lack of visual proximity of the data. This causes the user to have to constantly shift contexts which in turn affects their attentional and working memory resources.

Solution

Some initial sketches were done to consider varying low-fidelity prototypes. The most convincing concept sketch involved combining the process history and process diagram into a unified display.

Some key features of the redesign can be seen in figure 5 below. The process events window serves as the source of truth as the process history. This is in contrast from the current application which does not include every event (i.e. process documents are in a separate tab). The process events can then be filtered and searched to quickly find events of interest.

To further indicate the strong coupling between the process events window and the process diagram, a block from the diagram can be selected which then filters process events to the relevant events which occurred in the block.

Reflection

Conducting the redesign process was very engaging and challenging. It was very hard initially to understand the different user journeys because the application itself can be very complicated. As a result, initial time was taken to understand the application itself.

For the presented iteration, next steps for the redesign might be to conduct usability tests on administrators or mock internal users without. The focus would be on observing how the user gains an understanding of an active process and whether the changes make this challenge more efficient and accurate. Feedback provided would help guide the next design iteration before implementation is considered.
